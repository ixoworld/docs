## Domain Configurations
Set the domain properties of digital twin entities.
<AccordionGroup>

  <Accordion title="Identifiers">
  * DID
  * Also known as: Human-readable identifiers
  </Accordion>

  <Accordion title="Controllers">
  * Entity with update permissions
  * Example: "did:ixo:entity:abc123..."
  </Accordion>
  
  <Accordion title="Services">
  * Associated services and endpoints
  * Example: Data ingestion service with endpoint "https://api.ixo.earth"
  </Accordion>
  
  <Accordion title="Resources">
  * Digital media, files, and data
  * Example: Verifiable Credential
  </Accordion>
  
  <Accordion title="Rights">
  * Permissions and access control
  * Example: Delegated Authorisation
  </Accordion>
  
  <Accordion title="Claims">
  * Assertions made by entities or about entities
  * Example: Emission reduction
  </Accordion>
  
  <Accordion title="Relationships">
  * Networks of entities in the system
  * Example: Projects deploy Assets, financed by Investments
  </Accordion>
  
  <Accordion title="Accounts">
  * Blockchain account addresses
  * Example: Ethereum Network Address
  </Accordion>

  <Accordion title="Metadata">
  * Verification status
  * Credentials
  * Validity Period
  * State (e.g. active, inactive)
  * Ownership
  * Creator DID
  </Accordion>

</AccordionGroup>

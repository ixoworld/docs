Kitchen Performance Tests measure and validate real-world fuel consumption in household cooking environments through direct observation.

## Quick Start

<CodeGroup>
  ```bash Submit KPT
  curl -X POST https://api.emerging.eco/v1/claims \
    -H "Authorization: Bearer YOUR_API_KEY" \
    -d '{
      "type": "KPTClaim",
      "householdId": "did:ixo:household/123",
      "fuelMeasurements": {
        "total": 11.9,
        "dailyLogs": [3.2, 2.8, 3.0, 2.9]
      }
    }'
  ```

  ```python
  from emerging import Client

  client = Client('YOUR_API_KEY')
  kpt = client.claims.create(
      type="KPTClaim",
      household_id="did:ixo:household/123",
      fuel_measurements={
          "total": 11.9,
          "daily_logs": [3.2, 2.8, 3.0, 2.9]
      }
  )
  ```
</CodeGroup>

## Required Parameters

<ResponseField name="type" type="string" required>
  Must be "KPTClaim"
</ResponseField>

<ResponseField name="householdId" type="string" required>
  DID of the household being tested
</ResponseField>

<ResponseField name="fuelMeasurements" type="object" required>
  Daily fuel consumption measurements
</ResponseField>

## Optional Parameters

<ResponseField name="cookingSessions" type="array">
  Details of individual cooking events
</ResponseField>

<ResponseField name="evidence" type="array">
  Photos or documentation links
</ResponseField>

## Validation Rules

<AccordionGroup>
  <Accordion title="Data Consistency">
    * Daily measurements must be present

    * Total must match sum of daily logs

    * Values must be positive numbers
  </Accordion>

  <Accordion title="Device Correlation">
    * IoT data (if available) must align with KPT logs

    * Significant deviations trigger alerts
  </Accordion>
</AccordionGroup>

## Response Format

```json
{
  "id": "kpt-123",
  "status": "verified",
  "credential": {
    "type": "KPTCredential",
    "issuer": "did:ixo:validator/456",
    "evidence": [
      "ipfs://QmevidenceformX12"
    ]
  }
}
```

## Error Codes

<ResponseField name="400" type="error">
  Invalid KPT data format
</ResponseField>

<ResponseField name="401" type="error">
  Unauthorized request
</ResponseField>

<ResponseField name="409" type="error">
  Conflicting measurement data
</ResponseField>

## Integration Examples

### Combining with IoT Data

```python
# Get KPT baseline
kpt_baseline = client.kpt.get_baseline("household-123")

# Compare with IoT readings
iot_data = client.devices.get_usage("device-456")
reduction = calculate_reduction(kpt_baseline, iot_data)
```

<Note>
  Always cross-reference KPT data with available IoT measurements for accuracy.
</Note>

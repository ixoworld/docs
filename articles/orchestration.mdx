## What orchestration means in IXO

Orchestration coordinates actions between humans, software services, and agent systems using explicit permissions and verifiable state.

## Core concepts

<AccordionGroup>
  <Accordion title="Capability-based authorization" icon="key">
    Access is scoped to specific actions and resources, so services can operate with minimal required permissions.
  </Accordion>
  <Accordion title="Multi-actor coordination" icon="people-group">
    Workflows can include users, organizations, devices, and automation services in one governed process.
  </Accordion>
  <Accordion title="Policy and revocation" icon="shield-check">
    Permissions are time-bound and revocable, enabling safer long-running automations.
  </Accordion>
</AccordionGroup>

## Architecture fit

- IXO Protocol anchors identity and rights.
- IXO Matrix supports stateful collaboration and event exchange.
- Agentic services execute delegated workflow steps under policy constraints.
